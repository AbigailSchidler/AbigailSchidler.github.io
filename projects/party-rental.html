Party Reservation Platform

A full-stack reservation platform for booking party rooms and activity-based amenities — with account management, time-slot scheduling, and reservation history tracking.

Overview

A full-stack reservation platform for booking party spaces and activity-based amenities — trampolines, bowling lanes, climbing walls, and roller skating. Users browse a central service catalog, explore individual service pages, and reserve time slots through a calendar-based availability interface.

Reservations are stored per account and surfaced through a transaction history dashboard, giving users a persistent record of past and upcoming bookings.

Workflow

The booking flow is structured as a linear sequence — each step builds on the previous and narrows the user toward a confirmed reservation:

  1. Browse the service catalog by venue or activity type
  2. View individual service detail pages with availability
  3. Select a time slot using the scheduling interface
  4. Confirm the reservation through the account system
  5. Review booking history from the account dashboard

Features

The platform covers the full arc of a reservation workflow — from discovery to confirmation to history:

  • Service catalog browsing across venue and activity types
  • Time-slot scheduling with calendar-based availability
  • Account-based booking storage and history view
  • Search and availability filtering across services
  • Time-range booking controls
  • Verified-user review system

Technical implementation

The backend is built with Node.js and Express, using SQLite for persistent storage of services, accounts, and reservations. Time-slot availability is managed server-side with conflict checking enforced at the database level.

The frontend handles search and filter interactions without full-page reloads. The reservation confirmation workflow validates input and updates availability state before persisting the booking to the user's account record.

Context

Built as a full-stack interaction design exercise, this project applied frontend layout and interaction principles to a real booking workflow, while also implementing the backend infrastructure needed to support persistent accounts, time-based scheduling, and data retrieval across sessions.
