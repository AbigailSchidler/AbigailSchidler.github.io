FillerBuster

FillerBuster is a speech-practice tool that analyzes recorded presentations, highlights filler words in context, and helps speakers understand why hesitation language appears in their speech.

Prototype complete
Next.js Gemini API Supabase React JavaScript

Problem
Practicing presentations alone makes it difficult to notice filler-word habits or track improvement across rehearsal attempts
Built with
Next.js, Gemini API, Supabase, structured JSON prompting
Key design goal
Preserve speaker agency by explaining why words are marked as fillers rather than presenting model output as unquestionable feedback

Workflow Overview

The FillerBuster workflow is designed around iterative rehearsal rather than a single-pass correction. Users upload a recording, receive a verbatim transcript with filler words highlighted in context, and can return to review and compare across multiple saved takes.

The pipeline steps are: upload recording → generate verbatim transcript → detect contextual filler words → review highlighted transcript → compare across saved takes.

Each saved recording is surfaced from the dashboard as a named take. Selecting a take opens the highlighted transcript view, where filler words are marked inline with contextual annotations explaining why each phrase was flagged.

Features

Verbatim transcription

Accurate transcripts preserve hesitation language exactly as spoken, so filler words remain visible in the record rather than being cleaned away by normalization.

Context-aware filler detection

Words are only flagged when functioning as fillers, not when they carry meaning in context. "Like" in a comparison and "like" as a hesitation marker are treated differently.

Explainable feedback

Highlights communicate why words were marked, helping users interpret model decisions rather than accepting them as opaque judgments. This keeps the feedback process transparent and verifiable.

Multi-take reflection workflow

Saved recordings allow speakers to review progress across multiple rehearsal attempts, supporting pattern recognition over time rather than single-session correction.

Technical Pipeline

FillerBuster uses a two-stage LLM workflow. First, uploaded audio is transcribed verbatim using the Gemini API. Then the transcript is passed through Gemini again with structured prompting that returns JSON segments preserving original punctuation while tagging filler phrases and their detection rationale.

This structured output enables efficient rendering of interactive transcript highlights in the React interface, with each flagged segment carrying the context needed to display its explanation inline.

Designing Transparent Feedback

Because filler detection depends heavily on context, the interface exposes the reasoning behind each highlighted word rather than treating model output as authoritative. Users see not just which words were flagged, but why — grounding the feedback in the specific phrasing that triggered it.

This keeps users in control of the feedback process and supports verification pathways in future scoring workflows, where user confirmation of flagged words could inform accuracy metrics.

Supporting Reflective Speech Practice

Instead of treating filler detection as a single-pass correction tool, FillerBuster is designed around iterative rehearsal. By preserving transcripts across sessions, the interface helps users recognize patterns in hesitation language — whether fillers cluster at transitions, under time pressure, or in particular topic areas — and track improvement over time.

Future Directions

Possible improvements include:

  • User verification of flagged filler words to improve model accuracy
  • Progress tracking and trend visualization across sessions
  • Scoring based on accepted corrections over time
  • Expanded analytics for speech pattern recognition
